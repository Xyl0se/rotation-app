# Phase IX — Release, Search & Binding Assistance

**Status:** Planned — begins with the NAS release gate

---

## Purpose

Phase IX converts the stabilized self-hosted foundation into a verified release and
then improves the two workflows most likely to become cumbersome as the collection
grows: finding an album and resolving non-obvious folder bindings.

## Sprints

1. [Sprint 77 — Production Acceptance & Release Closure](./Sprint-77.md)
2. [Sprint 78 — Library Findability](./Sprint-78.md)
3. [Sprint 79 — Binding Candidate Review](./Sprint-79.md)

Sprint 79 is conditional on production examples showing that normalized matching
plus manual binding is insufficient. It may be postponed without blocking Phase X.

## Explicit Exclusions

- Existing Library perspectives are not rebuilt as filters.
- Matching proposals never auto-confirm uncertain bindings.
- Cross-browser synchronization is not introduced as hidden scope.
- Platform work does not begin before the remaining canonical browser data has an
  explicit server migration plan.
